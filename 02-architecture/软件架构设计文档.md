# 软件架构设计文档 (SAD)

> 版本：1.0 | 日期：2026-07-22 | 项目：12306旅客管理系统

## 1. 引言

### 1.1 文档目的
本文档描述12306旅客管理系统的软件架构，包括架构风格、技术选型、模块划分和部署方案。

### 1.2 架构概述
系统采用**前后端分离**的B/S架构。前端为Vue 3单页应用（SPA），后端为Spring Boot RESTful API服务，数据层使用MySQL + Redis。

## 3. 技术选型

| 层次 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 前端框架 | Vue 3 | 3.4+ | 渐进式框架，Composition API |
| 前端语言 | TypeScript | 5.3+ | 类型安全 |
| UI组件库 | Element Plus | 2.4+ | 成熟的Vue 3组件库 |
| 构建工具 | Vite | 5.0+ | 快速HMR |
| 后端框架 | Spring Boot | 3.2 | Java生态标准 |
| ORM | MyBatis-Plus | 3.5+ | Lambda查询，代码生成 |
| 数据库 | MySQL | 8.0 | 关系型，成熟稳定 |
| 缓存 | Redis | 7.0+ | 高性能KV存储 |
| 认证 | JWT | - | 无状态令牌 |

> 详细技术选型论证参见 `02-architecture/adr/` 目录下的架构决策记录。

## 4. 模块划分

### 4.1 后端模块

```
com.railway
├── controller/     # REST控制器层
│   ├── TrainController       # 车票查询
│   ├── PassengerController   # 乘车人管理
│   └── StationController     # 车站查询
├── service/        # 业务服务层
│   ├── TrainService / TrainServiceImpl
│   ├── PassengerService / PassengerServiceImpl
│   ├── StationService / StationServiceImpl
│   └── PassengerPrivacyService    # 隐私脱敏
├── mapper/         # 数据访问层（MyBatis-Plus）
│   ├── TrainMapper, TrainStopMapper, StationMapper
│   ├── PassengerMapper, UserMapper
│   └── OrderMapper, OrderItemMapper, SeatTypeMapper
├── entity/         # 实体类
│   ├── Train, TrainStop, Station
│   ├── Passenger, User, Order, OrderItem
│   └── SeatType
├── dto/            # 数据传输对象
│   ├── TrainQueryDTO
│   └── PassengerDTO
├── vo/             # 视图对象
│   └── TrainVO
├── common/         # 通用类
│   ├── Result      # 统一响应格式
│   └── IdType      # 证件类型枚举
└── config/         # 配置类
    └── SecurityConfig  # Spring Security + JWT配置
```

frontend/src/
