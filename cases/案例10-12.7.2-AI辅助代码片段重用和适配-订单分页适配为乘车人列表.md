# 案例10：AI辅助代码片段重用和适配——订单分页适配为乘车人列表

> **适用章节**：12.7.2 AI辅助代码片段重用和适配
> **工具形式**：ChatGPT / Copilot Chat / Cursor 代码改写
> **案例功能**：乘车人管理
> **来源文档**：12306_AI辅助软件开发案例_10-13章_全量补齐版

---

## 1. 案例背景

软件开发中常需复用已有代码。12306项目中 [OrderServiceImpl.getOrderList()](../../backend/src/main/java/com/railway/service/impl/OrderServiceImpl.java) 已实现分页查询+DTO转换模式，而乘车人列表查询（[PassengerServiceImpl](../../backend/src/main/java/com/railway/service/impl/PassengerServiceImpl.java)）需要类似的分页逻辑，但有额外要求：权限过滤、逻辑删除过滤、敏感字段脱敏。

---

## 2. 设计思想

AI辅助代码适配时，要求它说明"复用点"和"修改点"，避免机械替换变量名却遗漏业务差异（如脱敏、权限、逻辑删除）。

---

## 3. 工具输入 / 提示词输入

```
### 任务描述
请将下面的"订单列表分页查询"代码适配为"乘车人列表分页查询"。

### 原代码（来自OrderServiceImpl.getOrderList）
```java
public List<OrderVO> getOrderList(Long userId, Integer status) {
    List<Order> orders;
    if (status != null) {
        orders = this.lambdaQuery()
                .eq(Order::getUserId, userId)
                .eq(Order::getOrderStatus, status)
                .orderByDesc(Order::getCreateTime)
                .list();
    } else {
        orders = this.lambdaQuery()
                .eq(Order::getUserId, userId)
                .orderByDesc(Order::getCreateTime)
                .list();
    }
    return orders.stream()
            .map(order -> {
                List<OrderItem> items = orderItemMapper.selectList(
                    new LambdaQueryWrapper<OrderItem>()
                        .eq(OrderItem::getOrderId, order.getId())
                );
                return convertToVO(order, items);
            })
            .collect(Collectors.toList());
}
```

### 新业务要求
1. 根据userId查询该用户的乘车人列表（从passengers表）
2. 不返回已逻辑删除的乘车人（status字段或is_deleted字段）
3. 证件号码和手机号在返回前必须脱敏（调用PassengerPrivacyService）
4. 保留分页结构（Page + PageResult模式）
5. 返回PassengerDTO而非Passenger实体
6. 支持按姓名、证件类型筛选（可选参数）

### 项目实体参考
Passenger: id, userId, name, idType, idCard, passengerType,
           phone, nationality, birthDate, gender, expireDate, email,
           isDefault, isVerified, createTime, updateTime

### 输出要求
1. 适配后的完整方法代码
2. 说明复用点和修改点
```

## 4. AI输出 / 工具输出示例

**适配后代码**：

```java
public PageResult<PassengerDTO> queryPassengers(
        Long userId, String keyword, String idType, int pageNo, int pageSize) {

    // 构建查询条件
    LambdaQueryWrapper<Passenger> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(Passenger::getUserId, userId);

    // 条件筛选：姓名模糊搜索
    if (keyword != null && !keyword.isBlank()) {
        wrapper.like(Passenger::getName, keyword);
    }
    // 条件筛选：证件类型
    if (idType != null && !idType.isBlank()) {
        wrapper.eq(Passenger::getIdType, idType);
    }

    // 排除逻辑删除的乘车人
    wrapper.eq(Passenger::getStatus, 0);

    // 排序：默认乘车人优先 → 添加时间倒序
    wrapper.orderByDesc(Passenger::getIsDefault)
           .orderByDesc(Passenger::getCreateTime);

    // 分页查询
    Page<Passenger> page = new Page<>(pageNo, pageSize);
    Page<Passenger> resultPage = passengerMapper.selectPage(page, wrapper);

    // 实体→脱敏DTO
    List<PassengerDTO> records = resultPage.getRecords().stream()
            .map(passenger -> PassengerDTO.from(
                    passenger,
                    privacyService.maskIdCardNo(passenger.getIdCard()),
                    privacyService.maskPhone(passenger.getPhone())
            ))
            .collect(Collectors.toList());

    return new PageResult<>(records, resultPage.getTotal());
}
```

**复用点**：

| 复用的元素 | 来源 | 说明 |
|-----------|------|------|
| `LambdaQueryWrapper` 构建模式 | OrderServiceImpl | 链式条件构建，`.eq()` / `.orderByDesc()` |
| 可选筛选参数模式 | OrderServiceImpl 的 `Integer status` | `if (param != null) wrapper.eq(...)` |
| Stream + map + collect 转换模式 | OrderServiceImpl | 实体列表 → DTO列表 |
| 按创建时间倒序排列 | OrderServiceImpl | `.orderByDesc(Entity::getCreateTime)` |

**修改点**：

| 修改项 | 原代码 | 新代码 | 原因 |
|--------|--------|--------|------|
| 实体类型 | `Order` | `Passenger` | 不同业务实体 |
| 查询方式 | `this.lambdaQuery()` (ServiceImpl基类) | `passengerMapper.selectPage()` (显式Mapper) | 需要MyBatis-Plus分页 |
| 筛选条件 | `orderStatus` | `keyword` (姓名) + `idType` (证件类型) | 乘车人列表的筛选需求不同 |
| 逻辑删除过滤 | 无 | `.eq(Passenger::getStatus, 0)` | 乘车人需要逻辑删除 |
| DTO转换 | `convertToVO(order, items)` | `PassengerDTO.from(passenger, maskedIdCard, maskedPhone)` | 需注入脱敏结果 |
| 返回类型 | `List<OrderVO>` | `PageResult<PassengerDTO>` | 列表需要分页 |
| 排序优先级 | 仅 `createTime DESC` | `isDefault DESC` → `createTime DESC` | 默认乘车人排最前 |
| 脱敏 | 无 | `privacyService.maskIdCardNo()` / `maskPhone()` | 乘车人为敏感数据 |

## 5. 使用建议

1. **AI适配后要检查遗漏**：AI可能漏掉权限校验（只能查自己的乘车人）、状态过滤（排除已删除）等业务规则
2. **复用点→修改点对照表很有用**：可写入Code Review文档，帮助评审查人员理解变更意图
3. **DTO的from方法要干净**：`PassengerDTO.from()` 接收已脱敏的数据，避免DTO内部依赖PrivacyService
4. **分页查询注意性能**：若乘车人数量很大（接近30），分页收益有限；若支持批量导入（企业用户），分页就很重要
