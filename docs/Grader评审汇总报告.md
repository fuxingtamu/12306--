## Harness Engineering流程总结

```
┌─────────────────────────────────────────────────────────────┐
│                    Generator + Grader 流程                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Phase 1: Generator生成                                     │
│   ├── 输入：User Story / API需求 / Schema需求              │
│   ├── 输出：初始版本                                        │
│   └── 效率：快                                              │
│                                                             │
│   Phase 2: Grader评审                                       │
│   ├── 评分维度：完整性、正确性、一致性、可执行性              │
│   ├── 阈值：≥80分通过                                       │
│   └── 输出：问题清单 + 评分报告                             │
│                                                             │
│   Phase 3: 修正迭代                                         │
│   ├── 问题修复：Generator生成修正版                         │
│   ├── 再次评审：Grader确认                                  │
│   └── 输出：修正版文档                                      │
│                                                             │
│   Phase 4: 归档保留                                         │
│   ├── 原始版：保留用于对比                                  │
│   └── 修正版：用于后续开发                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
