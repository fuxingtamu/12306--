## 1. 案例背景

乘车人管理功能中，证件号码和手机号不能直接展示给前端。证件号脱敏属于局部、规则明确、上下文较少的小函数，最适合使用IDE中的AI代码补全完成。项目中脱敏逻辑需要支持8种证件类型的不同长度格式。

对应项目文件：前端的 [Passengers-Mobile.vue](../../frontend/src/views/Passengers-Mobile.vue) 中已实现前端脱敏展示，但后端缺少统一的脱敏服务。

---

## 2. 设计思想

本案例不是完整提示词问答，而是在代码中通过 Javadoc 注释写明脱敏规则，让AI根据当前文件上下文自动补全方法实现。这更符合实际编码场景——开发者写好注释描述业务规则，AI完成实现。

## 4. AI输出 / 工具输出示例

IDE中AI（如Copilot/GitHub Copilot Chat）根据 Javadoc 注释自动生成以下方法体：

```java
public String maskIdCardNo(String idCardNo) {
    // 规则1: 空值/空白处理
    if (idCardNo == null || idCardNo.isBlank()) {
        return "";
    }

    int length = idCardNo.length();

    // 规则2: 长度 <= 4
    if (length <= 4) {
        return "****";
    }

    // 规则3: 长度 <= 8，保留前2后2
    if (length <= 8) {
        return idCardNo.substring(0, 2)
                + "****"
                + idCardNo.substring(length - 2);
    }

    // 规则4: 长度 > 8，保留前4后4
    return idCardNo.substring(0, 4)
            + "********"
            + idCardNo.substring(length - 4);
}
```

**补充：手机号脱敏方法**（同样通过Javadoc注释触发AI补全）：

```java
/**
 * 对手机号进行脱敏。
 * 规则：
 * 1. null或空白 → 返回空字符串
 * 2. 11位手机号 → "138****8000" (保留前3后4)
 * 3. 非11位 → 仅保留首尾各1位，中间全星号
 *
 * @param phone 原始手机号
 * @return 脱敏后的手机号
 */
public String maskPhone(String phone) {
    if (phone == null || phone.isBlank()) {
        return "";
    }

    int length = phone.length();

    if (length == 11) {
        // 中国大陆手机号：138****8000
        return phone.substring(0, 3)
                + "****"
                + phone.substring(7);
    }

    // 非标准长度：只保留首尾各1位
    return phone.charAt(0)
            + "*".repeat(Math.max(0, length - 2))
            + phone.charAt(length - 1);
}
```

## 5. 使用建议

1. **Javadoc要写清楚规则**：空值处理、边界长度、示例，足够详细AI才能准确补全
2. **代码补全≠不用测试**：补全后立即执行预期结果速查验证
3. **统一脱敏入口**：项目中所有证件号/手机号展示都调用此Service，避免4个前端页面各自实现不同规则
4. **注意substring边界**：`length - 2` 和 `length - 4` 要确保索引合法
5. **Java 11+ 的 `repeat()`**：`"*".repeat(n)` 比 for循环拼接更简洁，但要注意项目Java版本
