# 12306旅客管理系统开发计划

## 一、项目背景

### 1.1 项目概述
开发一个功能完整的12306旅客管理系统，功能范围完全对标现有12306官方APP，包括用户管理、旅客管理、车票预订、订单管理等核心模块。

### 1.2 技术栈选型
参照12306实际技术架构，采用以下技术栈：

**前端技术栈**
- Vue 3 + TypeScript
- Element Plus UI组件库
- Pinia 状态管理
- Vue Router 路由管理
- Axios 网络请求
- Vite 构建工具

**后端技术栈**
- Java 17
- Spring Boot 3.x
- Spring Security + JWT 认证
- MyBatis-Plus ORM
- MySQL 8.0 数据库
- Redis 缓存

**部署方案**
- Gitee 代码仓库
- Gitee Pages 前端部署
- 云服务器后端部署

## 二、功能模块规划

| 功能 | 描述 |

### 2.4 购票模块
| 功能 | 描述 |
|------|------|
| 选座购票 | 选择座位类型和具体座位 |

| 订单确认 | 确认订单信息 |

### 2.5 订单管理模块
| 功能 | 描述 |

| 退票功能 | 申请退票、退票规则 |

### 2.6 候补购票模块
| 功能 | 描述 |
|------|------|
| 候补下单 | 无票时提交候补需求 |
| 候补状态 | 查看候补进度 |

### 2.7 其他功能
| 功能 | 描述 |
|------|------|

## 三、数据库设计

### 3.1 核心数据表
```
- users (用户表)
- passengers (旅客表)
- stations (车站表)
- trains (车次表)
- train_stops (经停站表)
- tickets (车票表)
- orders (订单表)
- seat_types (席别表)
- refunds (退票记录表)
- changes (改签记录表)
- waiting_list (候补订单表)
```

### 3.2 数据字典
详细数据库设计文档将在实现阶段完善。

### 3.3 数据来源说明
- **车站数据**：使用全国铁路真实车站数据（约3000+车站），包含车站代码、名称、所属城市、拼音缩写等
- **车次数据**：基于真实线路创建模拟车次，包含常见G/D/C/Z/T/K字头列车
- **线路数据**：覆盖主要高铁线路（京沪、京广、沪昆等）和普速线路
