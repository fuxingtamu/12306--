# 软件需求规格说明书 (SRS)

> 版本：1.0 | 日期：2026-07-22 | 项目：12306旅客管理系统

## 1. 引言

### 1.1 项目背景
12306旅客管理系统是一个功能完整的火车票查询与购票平台，为用户提供车票查询、乘车人管理、订单管理等功能。系统采用前后端分离架构，支持Web端访问。

### 1.2 适用范围
本文档定义系统的功能需求和非功能需求，作为后续设计、开发和测试的依据。

## 2. 功能需求

### 2.1 车票查询 (F-TQ-001)
**优先级**：P0（核心功能）

**描述**：用户输入出发地、目的地、出发日期后，系统查询符合条件的车次列表，展示每条车次的车次号、出发/到达站、出发/到达时间、历时、各席别余票数量和对应票价。

**输入**：出发站名称、到达站名称、出发日期（必填）；车次类型、出发时段、是否只看有票（可选筛选条件）

**输出**：车次列表，每条包含车次编号、车次类型、出发/到达站名、出发/到达时间、历时、各席别余票及票价

**业务规则**：
- 出发日期不得早于当前日期
- 支持按车次类型筛选（G高速/D动车/K普速等）
- 支持"只看有票"筛选：仅展示至少有一个席别余票>0的车次
- 默认按出发时间升序排列

### 2.2 乘车人管理 (F-PM-001)
**优先级**：P0（核心功能）

**描述**：用户可添加、修改、删除、查询乘车人信息。乘车人用于购票时快速选择乘客。

**子功能**：

| 功能 | 端点 | 描述 |
|------|------|------|
| 添加乘车人 | POST /api/passengers | 新增乘车人，含姓名、证件类型、证件号、乘客类型、手机号等 |
| 修改乘车人 | PUT /api/passengers/{id} | 修改已有乘车人信息 |
| 删除乘车人 | DELETE /api/passengers/{id} | 删除乘车人（逻辑删除） |
| 查询列表 | GET /api/passengers | 分页查询当前用户的乘车人列表 |
| 查询详情 | GET /api/passengers/{id} | 查询单个乘车人详细信息 |
| 设为默认 | PUT /api/passengers/{id}/default | 将指定乘车人设为默认乘车人 |

**业务规则**：
- 每个用户最多30名乘车人
- 同一用户不可重复添加相同证件号码的乘车人
- 删除乘车人前需检查是否存在未完成订单（待支付/已支付未出行），如有则不允许删除
- 证件号码和手机号在接口返回时必须脱敏
- 支持8种证件类型：身份证、港澳居住证、台湾居住证、外国人永居证、港澳通行证、台湾通行证、中国护照、外国护照

### 2.3 车站查询 (F-SQ-001)
**优先级**：P1

**描述**：提供车站列表查询，供前端站名选择器使用。支持按城市名或站名模糊搜索。

### 2.4 用户认证 (F-UA-001)
**优先级**：P0

**描述**：用户注册、登录、JWT令牌管理。

## 3. 非功能需求

### 3.1 性能
- 车票查询接口响应时间 ≤ 1秒（P95）
- 支持至少100并发查询请求

### 3.2 安全
- 用户认证使用JWT令牌机制
- 证件号、手机号等敏感信息在传输和展示时脱敏处理
- API接口需认证后访问（除登录/注册外）
- 密码使用BCrypt加密存储

### 3.3 可用性
- 系统可用性 ≥ 99.5%

### 3.4 技术约束
- 后端：Java 17 + Spring Boot 3.2 + MyBatis-Plus + MySQL + Redis
- 前端：Vue 3 + TypeScript + Element Plus + Vite
- 数据库：MySQL 8.0
- 缓存：Redis
