# 测试计划

> 版本：1.0 | 日期：2026-07-22 | 项目：12306旅客管理系统

## 1. 测试目标

验证12306旅客管理系统的功能正确性、业务规则完整性和安全隐私合规性。

## 2. 测试范围

| 功能模块 | 测试重点 | 优先级 |
|---------|---------|--------|
| 车票查询 | 查询条件校验、多表关联查询、筛选排序、只看有票逻辑 | P0 |
| 乘车人管理 | CRUD操作、重复证件号检查、30人上限、删除约束、脱敏 | P0 |
| 车站查询 | 列表查询、模糊搜索 | P1 |
| 用户认证 | 登录/注册、JWT令牌验证 | P0 |

## 3. 测试维度

| 维度 | 说明 |
|------|------|
| 正常路径 (Happy Path) | 合法输入下的预期行为 |
| 边界值 (Boundary) | 29/30/31人边界、空字符串、最大长度 |
| 异常路径 (Exception) | 无效输入、重复证件号、未完成订单删除 |
| 安全隐私 (Security) | 脱敏验证、未认证访问、JWT过期 |

## 4. 测试用例清单（代表性）

### 4.1 乘车人管理

| 用例编号 | 测试目标 | 输入/条件 | 预期结果 | 类型 |
|----------|---------|----------|---------|------|
| TC-PA-01 | 正常添加乘车人 | 完整有效信息 | 添加成功 | Happy |
| TC-PA-02 | 重复证件号拒绝 | 已有证件号 | 返回错误 | Exception |
| TC-PA-03 | 30人上限边界 | 已有29人，添加第30人 | 成功 | Boundary |
| TC-PA-04 | 超30人上限 | 已有30人，添加第31人 | 返回错误 | Boundary |
| TC-PA-05 | 未完成订单不可删 | 乘车人有PENDING订单 | 返回错误 | Exception |
| TC-PA-06 | 证件号脱敏 | 查询列表 | idCard为脱敏格式 | Security |
| TC-PA-07 | 手机号脱敏 | 查询列表 | phone为138****8000格式 | Security |

### 4.2 车票查询

| 用例编号 | 测试目标 | 输入/条件 | 预期结果 | 类型 |
|----------|---------|----------|---------|------|
| TC-TQ-01 | 正常查询 | 有效站名+日期 | 返回车次列表 | Happy |
| TC-TQ-02 | 过去日期拒绝 | 出发日期早于今天 | 返回错误 | Exception |
| TC-TQ-03 | 只看有票筛选 | onlyAvailable=true | 仅返回有余票车次 | Happy |
| TC-TQ-04 | 无匹配路线 | 不存在出发站 | 返回空列表+提示 | Exception |
| TC-TQ-05 | null值防御 | availableCount=null | 不抛异常 | Boundary |
